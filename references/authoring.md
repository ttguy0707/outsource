# 作者流程 · 新增自定义场景模板

用户要的场景库里没有时，按此把它沉淀成一个可复用的 `scenarios/<name>.md`（含一段五段式委派提示词，报告格式内嵌）。

## 步骤

1. **收原始诉求**：用户一两句话说清这个场景干嘛、在什么栈上。
2. **产澄清问题**：把下面的「澄清问题清单」按本场景裁剪后发给用户填。**能从诉求/上下文合理推断的，自己填默认值并标注，只问真拿不准的**——别把全部甩给用户。
3. **回填模板**：用用户答案填「场景文件骨架」，写到 `references/scenarios/<场景名>.md`。
4. **登记索引**：在 `references/scenarios/index.md` 的现有场景表加一行。
5. **自检**：过一遍「质量闸」。

## 澄清问题清单（裁剪后发给用户填）

```
请帮我把这个新场景定清楚（不确定的留空，我来补默认）：

1. 场景名 + 一句话适用：什么时候该用它？
2. 领域/角色：目标 session 该被定位成哪个领域的专家？（如 构建/数据/日志/运维/ML 数据…）
3. 背景/意图：这件事要确认或拿到什么？（让目标 session 理解，不是机械跑）
4. 技术栈/工具：用什么跑？（如 mvn / npm / python+pandas / sql / curl / kubectl …）
5. 工作流：要它依次做哪几步？每步尽量给命令或动作 + 这步的「目的」。
   - 步骤1（命令 / 目的）：
   - 步骤2：
   - 步骤3：
6. 护栏：是不是纯只读？有没有会改文件/数据/发写请求的风险？要明令禁止什么？
7. 哪些信息必须逐字原文回？（版本号/报错/命中行/符号…，防低配模型概括或编造）
8. 报告要回什么：结论形态（PASS/FAIL / 数值 / 是否 / 列表 / 表格）+ 还要哪些字段和证据？
9. 常见卡点：这类活常卡在哪？（网络/权限/缺库/超时/版本不存在…）
```

## 场景文件骨架（回填后写入 scenarios/<name>.md）

```
# 场景 · <场景名>

**适用**：<一句话；什么时候用>。

## 委派提示词（整段发给目标 session，已内嵌产出/报告格式）
\`\`\`
# 角色
你是<领域>专家。本次任务：<一句话目标>。

# 背景（帮你理解在找什么）
- 对象：<路径/栈/版本>
- 为什么做：<意图>

# 步骤（按序，每步留真实输出作证据）
1. <命令>　目的：<…>
2. <命令；X 不行就用 Y>　目的：<…>

# 铁律
- <只读/不改/不提交/脱敏…具体护栏>。
- <哪些信息>**逐字原文**回，禁改写补全简写美化。
- 查不到＝如实说查不到，绝不编造。
- 不确定/歧义 → 标出来，不猜。
- 只报事实与证据，不下主观结论。
- 卡点<列举>如实写，别反复重试，别扩范围。

# 产出（你按下面格式回传）
## <场景名> 报告
1. 结论：<PASS/FAIL/数值/是否>
2. <字段二>：__
3. <字段三 · 证据原文>：
   \`\`\`
   <粘贴>
   \`\`\`
4. 卡点：<无则写"无">
\`\`\`
```

## 质量闸（落盘前自检）

- [ ] 角色定位成「**<领域>专家**」+ 直接交代「本次任务：…」（不写"助手/替谁干活"）
- [ ] **背景/意图**写清（低配模型靠这个理解在找什么）
- [ ] 提示词**自包含**（路径/栈/版本/背景都给了，目标 session 无记忆）
- [ ] 每步是**可直接跑的命令** + 附「目的」，关键步有替代项
- [ ] 铁律齐：护栏(只读/脱敏) + **逐字原文** + **查不到说查不到** + 不确定标注 + 只报事实不下结论 + 卡点如实报
- [ ] **报告格式内嵌在"产出"段**（结论先行 + 要证据 + 卡点栏）
- [ ] 已在 `index.md` 登记一行
