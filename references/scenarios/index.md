# 场景模板索引

每个研发场景一个独立 md，含「适用 + 提示词 + 报告模板」，可直接抄改。通用骨架与检查清单见 `../templates.md`。

## 现有场景

| 场景 | 文件 | 一句话 |
|---|---|---|
| 构建 / 编译验证 | `build-compile.md` | 跑构建，报 PASS/FAIL + 错误原文 |
| 依赖 / 版本 / 符号(FQN)解析 | `dependency-symbol.md` | 查可用版本、从制品挖类全名与签名 |
| 数据查询 / 采集 | `data-query.md` | 只读查询 DB/接口/文件，原值返回 |
| 日志 / 报错排查 | `log-debug.md` | 按 pattern grep 日志，只报命中 |
| 环境 / 连通性探测 | `env-probe.md` | 探端口/服务/接口可达性 |
| 深度学习 / 数据集核查 | `dataset-inspect.md` | 看数据集规模/分布/标签/坏样本 |

## 怎么扩展（加新场景）

走作者流程 `../authoring.md`：收原始诉求 → 出澄清问题让用户填（工作流+护栏+报告字段+卡点）→ 据答案新建 `<场景名>.md`（适用/提示词/报告模板三段）→ 在上表登记一行 → 过质量闸。
守住通用约束：目标 session 无记忆（提示词自包含）、只读不改、结论先行、要证据、卡点如实报。

> 任何研发场景都能进：如 性能压测/benchmark、单测回归跑、notebook 执行、模型评测、API 契约比对、镜像构建、迁移脚本试跑 等——需要时照此新增。
